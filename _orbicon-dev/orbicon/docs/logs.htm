Logs

Orbicon framework automatically records events in a certain scope in order to provide an audit trail that can be used to diagnose problems.

Orbicon can track up to four types of logs. These are Debug, Error, PHP and System logs. System administrators can easily active or deactivate tracking of these logs adjusting the options available in logs.ini configuration file. Logs.ini is located in site/mercury directory.

Logs.ini options
These are the logs.ini configuration file options available to administrators. Comments start with ';'

  • errors - enable / disable error logging. Defaults to 1 (true).
  • system - enable / disable system logging. Defaults to 0 (false).
  • debug - enable / disable debug logging. Defaults to 0 (false).
  • php - enable / disable PHP error logging. Defaults to 0 (false).
  • path - path were log files will be saved. Enclose the value with double quotes ("). Defaults to site/mercury
  • merge - merge all logs into a single log file or keep them in their separate log files. Defaults to 0 (false).
  • log_rotator - select the default log archiving method. "size" will archive logs once they reach 16 Mb in size. "date" will archive a separate log for each day. Defaults to date.
