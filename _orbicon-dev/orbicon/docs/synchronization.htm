Synchronization

File and database synchronization is the process of making sure that two or more hosts contain the same up-to-date files and database information. If you add, change, or delete a file from local host, the synchronization process will add, change, or delete the same file at the remote host(s). The same applies for any database changes.

In a one-way sync as it is implemented in Orbicon Xtreme and Enterprise, also called mirroring, files are copied only from a "source" host to a "target" host, or hosts, but no files are copied back to the source host.

Although it is possible to synchronize contents manually, using Orbicon is both faster and less error prone. To start synchronization between single or multiple FTP/SSH2 servers (depending on your license) access System -> Synchronization
