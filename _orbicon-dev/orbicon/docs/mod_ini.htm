mod.ini

mod.ini is module's main configuration file. Without one, the module is considered invalid and will be ignored by the system. Below is an example of mod.ini's syntax. Two sections, module and about are required - a module builder could add additional module-specific sections.

Example mod.ini


Property Description Example
module section
name Module's name / title "Weather report"
small_icon Optional small icon for drop-down backend menu. The image must be stored in module's gfx folder and sized around 16×16px weather_s.png
medium_icon Optional medium icon for module's backend administration page. The image must be stored in module's gfx folder and sized around 37×37px weather_m.png
big_icon Optional big icon for drop-down backend menu. The image must be stored in module's gfx folder and sized around 80×80px weather_b.png
backend Main file loaded for backend administration. This PHP file's contents will be included into system's GUI. admin.weather.php
tab Backend menu section under which the module will be displayed. Not configuring this option means that the module will be accessed directy with a hyperlink - for example - http://server/?en=orbicon/mod/weather-report. Numeric values depend on section:
  • 1 - Content
  • 2 - Databases
  • 4 - Dynamic elements
  • 8 - Tools
  • 16 - CRM
  • 32 - Setting
  • 64 - System
 4
frontend Main file that will be loaded in the frontend. render.weather.php
runtime Type of frontend implementation. Value box will display frontend file's contents in a box (whose template mark is specified under template_mark option). Value page will display frontend file's contents as a full page. box
template_mark Template mark under which the module will be loaded. Required if the runtime value is set to box. WEATHER_REPORT
sync A list of module's SQL tables that will be synchronized with production servers during synchronization. Separate multiple tables with coma. orbx_mod_user_access, orbx_mod_user_data
Property Description Example
about section
author Module's author(s) "Pavle Gardijan (pavle.gardijan@orbitum.net)"
company Company that developed the module (if applicable) Orbitum ICT
version Module's software version 1.02b
date Date of module's creation 1/07/2006
updated Date of module's last update 06/04/2007
